# 第三阶段：高级交互与工作流

> **目标**：掌握 Claude Code 的效率技巧和进阶功能
> **预计时间**：3-4 天

---

## 学习清单

### 1. Plan Mode（计划模式）

核心流程：**探索 → 设计方案 → 用户确认 → 再执行**

Claude Code 进入 Plan Mode 后：
1. 先只读分析代码库（不能编辑）
2. 设计方案并写入计划文件
3. 用户审查计划
4. 批准后才开始执行

**检验标准**：用 Plan Mode 完成一个中等复杂度功能（如添加新 API 端点）

### 2. Agent / 子代理

| 概念 | 说明 |
|------|------|
| 用途 | 派生子任务并行处理 |
| 最多 | 7 个并行子代理 |
| 深度 | depth=1（子代理不能再开子代理） |
| 隔离 | `context: fork` 隔离上下文 |

常用场景：
- 同时调研 3 个独立问题
- 一个改前端，另一个改后端
- 一个写代码，另一个写测试

**检验标准**：让 Claude 同时调研 3 个独立问题

### 3. /ultrareview（云端代码审查）

- 云端多代理对抗分析
- 从安全、性能、可维护性等多维度审查
- 适合 PR 审查或关键代码检查

**检验标准**：对一次 PR 或代码修改运行 ultrareview

### 4. Computer Use（桌面操作）

- Claude 可直接操作本地 GUI 应用
- 打开浏览器验证页面
- 操作 IDE、设计工具等
- **注意**：Research Preview 阶段

**检验标准**：Claude 打开浏览器验证某个页面

### 5. /loop（定时循环）

语法：`/loop <间隔> <命令>`

示例：
- `/loop 5m /status` — 每 5 分钟检查构建状态
- `/loop 1h "检查测试结果"`

**检验标准**：设置一个有用的定时任务

### 6. /powerup（交互式教学）

内置交互式课程，直接在 CLI 中学习 Claude Code 功能。

**检验标准**：完成至少 1 个 powerup 课程

### 7. 常用快捷键

| 快捷键 | 功能 |
|--------|------|
| `Enter` | 提交输入 |
| `Ctrl+C` | 中断当前操作 |
| `↑/↓` | 历史命令 |
| `Ctrl+R` | 搜索历史 |
| `Esc` `Esc` | 撤销最近编辑 |
| `Shift+Tab` | 切换权限模式 |
| `Tab` | 补全/提示 |

**检验标准**：能不假思索使用 5 个以上快捷键
