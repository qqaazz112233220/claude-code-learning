# 第一阶段：基础操作与日常使用

> **目标**：能独立用 Claude Code 完成日常编码任务
> **预计时间**：2-3 天

---

## 1. 安装与启动

### 是什么
Claude Code 是一个终端里的 AI 编程助手，通过命令行使用。

### 操作
- **安装**：根据操作系统（Windows/Mac/Linux）执行安装命令
- **启动**：终端输入 `claude` 即可进入交互式会话
- **帮助**：输入 `/help` 查看内置帮助文档和快捷键列表

**检验标准**：能成功启动一个会话

---

## 2. 对话基础

### 是什么
用自然语言跟 Claude 对话来完成编程任务。不需要记特殊语法，像跟人说话一样就行。

### 工作流程
```
你描述需求 → Claude 理解并追问 → 你补充信息 → Claude 执行 → 你确认结果
```

### 关键点
- **描述需求**：清晰说明你想做什么，越具体越好
- **追问澄清**：Claude 遇到模糊的地方会主动提问，如实回答即可
- **接受/拒绝**：对 Claude 的输出可以说"行"直接接受，或者说"不对，换个方式"让它重来

### 例子
```
你：帮我把这个文件里的 console.log 全部删掉
Claude：（找到所有 console.log 并删除）
你：不错（确认）

你：帮我优化一下这个函数
Claude：你想优化什么方面？性能？可读性？
你：可读性
```

**检验标准**：完成一次完整的"描述需求 → Claude 实现 → 确认"流程

## 3. 文件操作

### 是什么
Claude Code 内置了文件读写工具，不需要你手动打开编辑器改文件。

### 三个核心工具

#### Read — 读取文件
- **能看什么**：源代码、图片（PNG/JPG等）、PDF（最多20页）、Jupyter Notebook
- **单次上限**：2000 行
- **大白话**：相当于你用 VS Code 打开文件看看里面写了啥

#### Write — 创建/覆盖文件
- **用途**：创建新文件，或者完全重写一个已有文件
- **大白话**：相当于"新建文件"或"另存为"

#### Edit — 精确修改（推荐！）
- **用途**：只替换文件中匹配到的部分，不改动其他内容
- **大白话**：比 Write 更安全——只改你想改的那几行，文件的其他部分不动

### 例子
```
你：把 index.html 的标题改成"我的网站"
Read → 找到 index.html 中 <title>原标题</title>
Edit → 把 "原标题" 替换成 "我的网站"
```

**检验标准**：让 Claude 修改一个现有文件并验证改动正确

---

## 4. Bash 命令

### 是什么
Claude 可以直接在终端里执行 shell 命令，不需要你手动复制粘贴去跑。

### 两种触发方式
1. **让 Claude 自己决定**：你说"装一下依赖"，Claude 自动执行 `npm install`
2. **`!` 前缀**：在消息开头加 `!`，后面的内容直接当 bash 命令执行，Claude 不介入

### 常用场景
| 场景 | 示例 |
|------|------|
| 安装依赖 | `npm install` / `pip install` |
| 运行测试 | `npm test` / `pytest` |
| Git 操作 | `git status` / `git diff` / `git commit` |
| 文件操作 | `ls` / `mkdir` / `cp` |

### 例子
```
你：帮我看看当前目录有什么文件
Claude：执行 ls，返回结果

你：!git status
（直接执行 git status，Claude 不参与）
```

**检验标准**：让 Claude 完成 `npm install && npm test`

---

## 5. 搜索代码

### 是什么
Claude Code 内置了搜索工具，不用离开终端就能在项目里找代码。

### 两个搜索工具

#### Grep — 按内容搜索
- **用途**：在文件内容中搜索指定的关键词或正则表达式
- **相当于**：ripgrep
- **例子**：`找到项目中所有用到 useState 的地方`

#### Glob — 按文件名搜索
- **用途**：通过通配符模式匹配文件名
- **相当于**：find 命令
- **例子**：`找所有 .test.tsx 结尾的文件`

### 例子
```
你：找到 utils/ 目录下所有带 "format" 的函数的定义
Claude：用 Grep 搜索 "function.*format" 和 "const.*format"，然后展示结果
```

**检验标准**：找到项目中一个特定函数的定义和所有调用处

## 6. 权限模式

### 是什么
控制 Claude 能做什么、需要问你什么的一套安全级别。按 `Shift+Tab` 循环切换。

### 五种模式（从严格到宽松）

| 模式 | 行为 | 什么时候用 |
|------|------|-----------|
| **plan** | 只读，不能编辑任何文件 | 分析问题、设计方案，先看不动手 |
| **default** | 敏感操作（改文件、跑命令）需要你确认 | 日常开发，安全第一 |
| **acceptEdits** | 文件编辑自动接受，其他操作需要确认 | 批量改文件时，省得一直点确认 |
| **auto** | 安全操作自动放行，危险操作拦截 | 信任的重复任务 |
| **bypassPermissions** | 所有操作自动放行，完全不问 | 完全信任的自动化场景 |

### 大白话
- **plan**：顾问模式——只动嘴不动手
- **default**：普通模式——每干一件"大事"都问你一下
- **acceptEdits**：信任模式——改文件不用问，但跑危险命令还是要问
- **auto**：半自动——安全的自动，不安全的拦截
- **bypassPermissions**：全自动——完全撒手让 Claude 干

**检验标准**：能解释各模式适用场景

## 7. Checkpoint / 撤销

### 是什么
Claude Code 在编辑文件前会自动备份当前版本，改错了可以一键还原。

### 操作方式
- **两次 `Esc`**：撤销最近一次文件修改
- **`/undo`**：和按两次 Esc 效果一样

### 底层机制
1. Claude 准备修改文件时，自动创建当前文件的快照（checkpoint）
2. 执行编辑操作
3. 你按 `Esc Esc` 或 `/undo` → 用快照还原文件

### 限制
- **只能撤销最近一次修改**，不能撤回到更早的版本
- 每轮编辑只有一个 checkpoint

### 例子
```
你：帮我在 config.js 里加一个配置项
Claude：修改了 config.js（自动创建了 checkpoint）
你：不对，改错了
你：按两次 Esc → config.js 回到修改前的状态
```

**检验标准**：让 Claude 改错文件后成功回退
