## 学习清单

### 1. 上下文成本意识

使用 `/usage` 命令分析：

| 指标 | 含义 |
|------|------|
| 缓存命中率 | prompt caching 效果，影响响应速度 |
| 子代理开销 | 多个子代理的 token 消耗 |
| 会话总 token | 当前会话累积了多少 token |
| 模型比例 | Opus/Sonnet/Haiku 各自的使用量 |

**优化策略**：
- 频繁使用的上下文移入 CLAUDE.md（自动缓存）
- 大文件用 Grep/Glob 而不是 Read 全文
- 长会话及时 `/clear` 或 `/compact`

**检验标准**：能诊断并优化一次高消耗会话

### 2. 模型选择策略

| 任务类型 | 推荐模型 | 原因 |
|----------|----------|------|
| 复杂架构设计 | Opus 4.7 | 最强推理能力 |
| 日常编码 | Sonnet 4.6 | 速度与质量平衡 |
| 简单格式化 | Haiku 4.5 | 低成本、快速 |
| 代码审查 | Opus 4.7 / UltraReview | 需要深度分析 |
| 批量重构 | Sonnet 4.6 + auto mode | 高效且安全 |

Effort 级别：`low` / `medium` / `high` / `xhigh`

**检验标准**：为不同任务正确选型

### 3. Agent Team（多代理协调）

实验性功能，多个子代理协作完成复杂任务：

```
主代理（协调者）
├── 子代理 A：分析需求，产出规格
├── 子代理 B：设计方案，产出架构
├── 子代理 C：实现代码
├── 子代理 D：编写测试
└── 子代理 E：审查结果
```

**检验标准**：让多个子代理协同完成一个复杂功能

### 6. 深度排错

常见问题排查路径：

| 问题 | 排查方向 |
|------|----------|
| Hook 不生效 | 检查 exit code、路径、执行权限 |
| MCP 连接失败 | 检查 `.mcp.json` 格式、环境变量 |
| 模型响应异常 | 检查上下文是否污染 → `/clear` |
| 权限提示过多 | 调整 settings.json 中的 allow/deny 规则 |
| 子代理不返回结果 | 检查 `context: fork` 是否正确 |

**检验标准**：独立排查并解决一次集成故障
