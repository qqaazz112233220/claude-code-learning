# 权限模式详解（参考文档）

> Claude Code 五种权限模式 + 细粒度权限控制的完整说明

---

## 为什么需要权限模式？

Claude Code 能做的事情很多——读文件、写文件、跑命令、装依赖、搜网络。有些操作是**不可逆的**（删文件、强制推送），有些只是**查看**。

权限模式就是让你一键切换 "Claude 能自作主张到什么程度"。

**切换方式**：按 `Shift+Tab` 循环切换，当前模式显示在终端底部。

---

## 五种模式详解

### 1. plan — 计划模式

**是什么**：只读模式。Claude 只能看文件、搜代码，不能创建/编辑任何文件，也不能执行 bash。

**底层**：工具集限制：
- ✅ 允许：Read、Glob、Grep、WebFetch、WebSearch
- ❌ 禁止：Write、Edit、Bash

**大白话**：只动眼不动手，出方案不改代码。

**啥时候用**：
- 接手不熟悉的项目，先探索代码结构
- 大改动前先出方案，你再审查
- 单纯搜索/分析，不用担心误改

**和 `/plan` 命令的关系**：`/plan` 自动切换到 plan 模式，方案批准后自动切回原模式。

**例子**：
```
你：/plan 帮我分析这个项目的目录结构
→ Claude 只读扫描，不能改任何文件
```

---

### 2. default — 默认模式（最常用）

**是什么**：敏感操作需要确认，低风险操作自动放行。

**底层**：按工具类型区别处理：
- 读文件/搜索 → 自动放行
- 写文件/改文件 → 弹框确认
- 执行 bash → 弹框确认
- 联网搜索 → 取决于 settings.json

弹框选项：
- **Allow**：这次允许
- **Allow for session**：本次会话内都允许
- **Always allow**：永久允许（写入 settings.json）
- **Deny**：拒绝

**大白话**：干啥都问一声，安全省心。

**啥时候用**：绝大多数日常开发。

**例子**：
```
你：帮我看下 package.json → 自动放行
你：把版本号改成 2.0.0 → 弹框确认 → 你点 Allow
```

---

### 3. acceptEdits — 自动接受编辑

**是什么**：文件编辑自动放行，bash 等操作仍需确认。

**底层**：
- Write、Edit → **自动允许**
- Bash → 仍需确认
- 读文件/搜索 → 自动放行

**大白话**：改文件不问，跑命令问。

**啥时候用**：批量改文件时（重构、重命名），省得每改一个点一次。

**和 default 的区别**：
```
default：每改一个文件 → 弹框 → 点 Allow → 继续
acceptEdits：改文件 → 直接改 → 改完告诉你
```

**例子**：
```
你：把 src/ 下所有 .js 改成 .ts
acceptEdits：逐个改，不弹框
default：改一个弹一次，100 个文件点 100 次
```

---

### 4. auto — 自动模式

**是什么**：内置安全规则判断操作风险，安全的自动放行，危险的操作拦截。

**底层**：启发式风险判断：
- ✅ **安全**（自动放行）：读文件、搜代码、列出目录、无害 git 操作
- ⚠️ **风险**（拦截）：`rm -rf`、`git push --force`、修改敏感配置
- 🤔 **中间地带**：安装包一般安全，不明来源的包拦截

**和其他模式的区别**：
- default：所有写操作都弹框
- acceptEdits：编辑不弹但命令弹
- auto：让系统自己判断危不危险

**大白话**：聪明的自动驾驶——自动判断要不要问你。

**啥时候用**：信任的重复任务，想留一道安全防线，又不想被弹框烦。

**例子**：
```
你：删掉 dist/ → 系统判断 rm -rf 有风险 → 拦截并确认
你：打包日志 → 系统判断 tar 没风险 → 自动执行
```

---

### 5. bypassPermissions — 绕过所有权限

**是什么**：所有操作直接执行，没有任何确认。

**底层**：彻底关闭权限检查层，所有工具一律自动放行。

**大白话**：完全信任，放手干。不设防。

**啥时候用**：
- 全自动化 CI/CD
- 你盯着屏幕看，不需要每步确认
- 深夜跑批量任务不想守着点确认

**风险**：误操作直接执行，没有缓冲。不建议日常开。

**例子**：
```
你：更新所有 mock 数据
Claude 直接找文件、改文件、保存，全程不说话
（改错了也不停，改完说"搞定了"）
```

## 各模式对比

| 模式 | 读文件/搜索 | 写文件/编辑 | Bash 命令 | 一句话 |
|------|:-----------:|:-----------:|:---------:|--------|
| **plan** | ✅ | ❌ 禁止 | ❌ 禁止 | 只动眼不动手 |
| **default** | ✅ | 弹框确认 | 弹框确认 | 干啥都问一声 |
| **acceptEdits** | ✅ | ✅ 自动 | 弹框确认 | 改文件不问，跑命令问 |
| **auto** | ✅ | 安全自动/危险拦截 | 安全自动/危险拦截 | 聪明的自动驾驶 |
| **bypassPermissions** | ✅ | ✅ | ✅ | 完全放手 |

## 细粒度权限控制

### `/allow` 和 `/disallow`

赋予或撤销某个工具的权限，写入 settings.json：

```
/allow Bash           # Claude 以后跑 bash 不再问你
/disallow WebSearch   # Claude 联网搜索必须先问你
```

### settings.json 中配置

```json
{
  "permissions": {
    "allow": ["Bash", "WebSearch"],
    "deny": []
  }
}
```

`allow` 列表里的工具在 **default 及以上模式**自动放行（plan 除外，plan 硬性只读）。

### 会话级临时权限

default 模式下弹框点 **Allow for session**，本次会话不再问，下个新会话恢复。

## 实践建议

| 场景 | 推荐模式 |
|------|---------|
| 刚接触 Claude Code | default |
| 探索新项目/分析代码 | plan |
| 批量重构/改大量文件 | acceptEdits |
| 重复性信任任务 | auto |
| 通宵跑自动化 | bypassPermissions（谨慎） |

**最佳实践**：平时 default，批量改切 acceptEdits，分析方案切 plan。不用一条模式走到黑。
